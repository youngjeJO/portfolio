■ 프로젝트 1: 물류센터 업무 통합 시스템 (DHL ILMS)
──────────────────────────────────────────────────────────────────────────────────

  기간:     2025.06 ~ 2025.09
  소속:     ㈜굿스플로 SCM팀
  규모:     중대형 (물류센터 현장 업무 시스템)
  역할:     프론트엔드 주도 개발
  기술스택: React, TypeScript, Zustand, Styled-components, SCSS, Axios, Git

  [프로젝트 설명]
  DHL 인천 물류센터의 출고, 검수, 프린트, 작업 관리 등 현장 업무를 효율적으로
  처리하기 위한 웹 기반 통합 물류관리 시스템. 바코드 스캔 기반의 실시간 데이터
  처리와 자동화된 워크플로우를 통해 물류 업무의 정확성과 효율성을 향상.

  [주요 업무]

  1. 바코드 스캔 기반 실시간 출고/검수 시스템
     - 바코드 리더기 연동을 통한 실시간 스캔 데이터 처리
     - 2단계 검수 프로세스: PID 스캔 → 운송장번호 스캔 → sheetNo 비교 검증
     - 실시간 상태 피드백: 스캔 성공/실패에 따른 시각적/청각적 피드백 제공
     - 디바운스 처리: 중복 스캔 방지를 위한 1초 디바운스 적용
     → 검수 정확도 향상 및 작업 시간 단축

  2. 상태 관리 아키텍처 설계 및 구현
     - Zustand 기반 전역 상태 관리 (useJobStore, useDetailStore,
       useConfirmStore, usePrintStore, useContractStore)
     - useShallow를 활용한 불필요한 리렌더링 방지
     - async/await 기반 API 연동 및 에러 핸들링
     → 복잡한 상태 관리 로직을 체계적으로 구조화하여 유지보수성 향상

  3. 환경설정 시스템 구축
     - 쿠키 기반 설정 저장: 프린터, 발송지, 배송사, 박스규격,
       유형별 색상/사운드 등
     - 설정값 영구 저장: 10년 만료 쿠키로 설정값 영구 보존
     - 유형별 커스터마이징: 일반/애플/B2C 유형별 색상 및 사운드 개별 설정
     - 사운드 미리듣기: 설정 모달에서 사운드 선택 시 즉시 재생 기능
     → 사용자별 맞춤 설정 지원으로 업무 효율성 향상

  4. 인증 및 세션 관리
     - 사용자 정보 자동 동기화: 토큰 갱신 시 mem_idx, user_idx 자동 쿠키 저장
     - 자동 로그아웃: 30분 비활성 시 세션 만료 모달 표시 및 리다이렉트
     - 세션 만료 모달: 사용자 친화적인 세션 만료 알림 UI 구현
     → 보안 강화 및 사용자 경험 개선

  5. 프린트 서비스 통합
     - 외부 프린트 서비스와의 통합 연동
     - 스캔 데이터 기반 자동 프린트 실행
     - 프린트 결과 피드백: 성공/실패에 따른 사운드 및 시각적 피드백
     - 설정 기반 프린트: 사용자 설정(프린터, 계약정보) 반영
     → 자동화된 프린트 프로세스로 업무 효율성 향상

  6. Wijmo Grid 기반 데이터 테이블
     - 고성능 그리드 구현: 대용량 데이터 처리 최적화
     - 실시간 데이터 필터링 및 정렬 기능
     - Excel 내보내기 기능
     → 직관적인 데이터 조회 및 관리 기능 제공

  7. UI/UX 개선
     - Styled-components 기반 디자인 시스템: 일관된 UI 컴포넌트 설계
     - 반응형 레이아웃, 애니메이션 효과, 접근성 향상
     → 사용자 친화적인 인터페이스로 학습 곡선 단축

  8. 유틸리티 및 공통 모듈 개발
     - 날짜 포맷, 쿠키 관리, 사용자 정보 관리, 에러 처리 유틸리티
     → 코드 재사용성 향상 및 유지보수성 개선

  [문제 해결 경험]
  - 다수 스토어 간 상태 동기화 문제 → useShallow 선택적 상태 구독으로 해결
  - 빠른 연속 스캔 시 중복 처리 → 디바운스(1초) 및 상태 기반 입력 제어
  - 쿠키 기반 설정 관리 → JSON 기반 객체 저장 및 초기값 자동 설정
  - 외부 프린트 서비스 비동기 연동 → Promise 기반 처리 및 에러 핸들링 강화

  [성과]
  - 기술: Zustand 기반 상태 관리 아키텍처 설계, 바코드 스캔 기반 실시간 시스템 구현
  - 비즈니스: 검수 정확도 향상, 작업 처리 시간 단축, 대규모 현장 시스템 안정화

  [기여 요약]
  프론트엔드 주도 개발, 바코드 스캔 기반 출고·검수 시스템 구현,
  Zustand 기반 상태 관리 아키텍처 설계, 외부 프린트 서비스 연동


■ 프로젝트 2: Cafe24 운송장 출력 시스템 & Web BFF
──────────────────────────────────────────────────────────────────────────────────

  기간:     2024.11 ~ 2025.06
  소속:     ㈜굿스플로 SCM팀
  규모:     중대형 (이커머스 물류·운송장 출력 실운영 시스템)
  역할:     프론트엔드 / BFF 설계 및 구현
  기술스택: React 18, TypeScript, Zustand, Axios, SCSS, ASP.NET Core,
            Cafe24 API, 이니시스 결제 API, Git

  [프로젝트 설명]
  Cafe24 입점 셀러 및 물류 운영자를 위한 운송장 출력 시스템. 복잡하게 분산된
  Cafe24 API 및 내부 시스템을 프론트엔드에서 사용하기 쉬운 형태로 통합하고,
  프론트엔드 요구사항을 기준으로 BFF 구조를 설계하여 운송장 출력, 주문·결제·배송
  흐름을 하나의 화면·API 흐름으로 단순화. 실운영 환경에서 성능·안정성·유지보수성
  개선이 목표.

  [주요 업무]

  1. 프론트엔드 중심 BFF(Service Layer) 설계
     - 화면 단위 요구사항을 기준으로 API 응답 구조 재정의
     - 여러 백엔드/외부 API를 단일 엔드포인트 형태로 집계(Aggregation)
     - 도메인별 서비스 모듈 분리 (Order / Payment / Contract / Alert 등)
     → 프론트엔드 컴포넌트 복잡도 감소 및 API 의존성 감소, 재사용성 향상

  2. React 기반 프론트엔드 구현
     - React 18 기반 SPA 개발
     - Zustand를 활용한 전역 상태 관리 구조 설계
     - API 응답을 프론트엔드 상태 구조에 맞게 정규화
     - 불필요한 리렌더링 방지를 위한 상태 분리 및 메모이제이션 적용
     → 화면 상태 관리 단순화 및 유지보수성, 가독성 개선

  3. 인증 및 세션 관리 (Frontend + BFF 협업)
     - Cafe24 OAuth 2.0 인증 흐름 연동
     - 쿠키 기반 세션 유지 및 토큰 자동 갱신 처리
     - 세션 만료 시 사용자 경험을 고려한 공통 처리 로직 구현
     → 세션 만료로 인한 사용자 이탈 감소 및 인증 로직의 일관성 확보

  4. 결제 및 외부 서비스 연동
     - 이니시스 결제 API 연동
     - 팝업 기반 결제 프로세스를 PostMessage로 제어
     - 결제 승인·환불·부분환불 흐름 처리
     - 결제 결과에 따른 UI 상태 동기화
     → 결제 흐름 안정화 및 프론트엔드 로직 단순화

  5. 데이터 흐름 및 성능 최적화
     - 병렬 API 호출(Promise.all / allSettled) 적용
     - 폴링 구조 개선 및 무한 스크롤 적용
     - 불필요한 API 호출 제거
     → 초기 로딩 시간 약 30% 개선, API 호출 횟수 약 60% 감소

  6. BFF 백엔드(Web BFF) 구현 — ASP.NET Core
     - ASP.NET Core 기반 BFF 서버 설계 및 구현
     - 프론트엔드 전용 API 제공
     - 주문·상품·배송 데이터를 UI 기준으로 그룹핑·정규화
     - 인쇄/송장 콜백 처리로 UI 상태 동기화
     - 요청/응답 로깅 및 예외 처리 구조 설계
     → 프론트엔드와 백엔드 간 책임 분리 명확화, 장애 분석 및 대응 용이

  [문제 해결 경험]
  - 화면 하나에 여러 API 직접 호출로 인한 복잡성
    → BFF 구조 도입, API 응답 재구성 및 단일 엔드포인트 집계
  - 인증/세션/에러 처리 화면별 불일치
    → OAuth 2.0 + 쿠키 세션 + 토큰 갱신을 공통 처리로 통합
  - 잦은 폴링·불필요 API 호출로 인한 로딩 속도 문제
    → 병렬 호출, 폴링 개선, 불필요 호출 제거 (30% 로딩 개선, 60% API 감소)
  - 결제·배송 상태 동기화 안정성
    → 이니시스 PostMessage 제어 + 송장 콜백으로 상태 동기화

  [성과]
  - 기술: BFF 서비스 레이어 설계, API 통합 및 데이터 정규화,
          인증(OAuth) 및 상태 관리 구조 개선,
          초기 로딩 속도 약 30% 개선, API 호출 횟수 약 60% 감소
  - 비즈니스: 주문·결제·배송 흐름 단순화, 실운영 성능·안정성·유지보수성 개선,
              프론트엔드 개발 속도 향상, 세션 만료 이탈 감소

  [기여 요약]
  프론트엔드 요구사항 기준 BFF 서비스 레이어 설계, API 통합 및 데이터 정규화,
  인증(OAuth) 및 상태 관리 구조 개선, SPA 전용 BFF API 구축(ASP.NET Core),
  주문·결제·배송 데이터 통합, 쿠키 기반 인증 및 토큰 갱신 처리

  [프로젝트 설명]
  쇼핑몰 운영자 및 관리자를 위한 이커머스 관리 시스템. 프론트엔드 요구사항을
  기준으로 BFF 구조를 설계하여 이커머스 운영 화면에서 필요한 데이터를 한 번에
  제공하고, 프론트엔드 복잡도를 낮추며 API 사용성과 유지보수성을 개선.
  인증·상태·에러 처리 방식을 일관되게 통합하여 운영 안정성을 향상.

  [주요 업무]

  1. 프론트엔드 개발 (GODO 운영 시스템)
     - 운영자용 이커머스 관리 화면 개발
     - 화면 단위 요구사항 정의 및 UI 구성
     - 상태 변화가 많은 화면에서의 데이터 흐름 관리
     - 사용자 액션에 따른 실시간 UI 상태 반영
     → 운영 화면 가독성 및 사용성 개선, 프론트엔드 로직 구조 단순화

  2. BFF(Service Layer) 설계 및 구현
     - 프론트엔드 요구사항 기준으로 API 응답 구조 재정의
     - 여러 API를 하나의 엔드포인트로 집계(Aggregation)
     - 도메인 단위 서비스 모듈 분리
     → 프론트엔드 API 호출 수 감소 및 화면 개발 생산성 향상

  3. 인증 및 세션 관리
     - 쿠키 기반 인증 구조 설계
     - 세션 유지 및 만료 처리 공통화
     - 인증 실패/만료 시 공통 UX 제공
     → 인증 관련 중복 코드 제거 및 사용자 경험 일관성 확보

  4. 데이터 가공 및 정규화
     - 백엔드 응답 데이터를 UI 기준으로 그룹핑·정규화
     - 프론트엔드에서 불필요한 데이터 처리 로직 제거
     → 프론트엔드 코드 복잡도 감소 및 유지보수성 향상

  5. BFF 백엔드 구현 (GODO BFF)
     - BFF 서버에서 프론트엔드 전용 API 제공
     - 요청/응답 로깅 및 예외 처리 구조 설계
     - 장애 발생 시 원인 분석이 가능한 구조 마련
     → 운영 중 이슈 대응 속도 개선, 프론트엔드–백엔드 책임 분리 명확화

  [문제 해결 경험]
  - 화면 구성을 위한 다수 API 개별 호출 문제
    → BFF 구조 도입, API 응답 재구성 및 집계
  - 화면별 인증/세션/에러 처리 불일치
    → 쿠키 기반 인증, 세션 유지/만료 공통화
  - 운영 중 장애 원인 추적 어려움
    → BFF 서버 요청/응답 로깅 및 예외 처리 구조 설계
  - 프론트엔드 데이터 처리 복잡도 증가
    → BFF 레이어에서 UI 기준 데이터 그룹핑·정규화 처리

  [성과]
  - 기술: BFF 구조 도입으로 API 사용성 개선, 데이터 가공 로직 감소,
          인증·에러 처리 일원화, 장애 대응·분석 용이성 확보
  - 비즈니스: API 호출 수 감소(네트워크 효율성 향상), 화면 개발 생산성 향상,
              운영 화면 사용성 개선, 이슈 대응 속도 개선

  [기여 요약]
  프론트엔드 요구사항 정의 및 화면 구현, BFF 구조 설계 및 API 재구성,
  인증·세션·데이터 흐름 설계, 운영 환경 기준 안정성 개선
